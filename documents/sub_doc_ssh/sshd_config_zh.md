

1. **ChallengeResponseAuthentication no**:
   - **作用**: 禁用挑战-响应式密码验证。
   - **使用场景**: 提高安全性，避免使用可能存在兼容性问题的验证方式。通常使用更安全的验证方法，如公钥认证。

2. **Kerberos 选项**:
   - **KerberosAuthentication no**: 禁用 Kerberos 验证。
   - **KerberosOrLocalPasswd yes**: 允许使用 Kerberos 或本地密码验证。
   - **KerberosTicketCleanup yes**: 在退出时清理 Kerberos 票据。
   - **KerberosGetAFSToken no**: 禁用获取 AFS 令牌。
   - **使用场景**: 适用于需要 Kerberos 的环境，通常在企业级网络中使用以实现单点登录。

3. **GSSAPI 选项**:
   - **GSSAPIAuthentication no**: 禁用 GSSAPI 验证。
   - **GSSAPICleanupCredentials yes**: 在会话结束时清理 GSSAPI 凭据。
   - **GSSAPIStrictAcceptorCheck yes**: 启用严格的接受者检查。
   - **GSSAPIKeyExchange no**: 禁用 GSSAPI 密钥交换。
   - **使用场景**: 适用于需要 GSSAPI 的环境，通常用于增强的安全验证和单点登录。

4. **UsePAM yes**:
   - **作用**: 启用 PAM（可插拔认证模块）进行认证、账户处理和会话处理。
   - **使用场景**: 允许更灵活的认证机制和账户管理。通过 PAM，可以集成各种认证方法和策略。适用于需要复杂认证配置的环境。

5. **AllowAgentForwarding yes**:
   - **作用**: 允许 SSH 代理转发。
   - **使用场景**: 当你使用 SSH 代理（如 `ssh-agent`）管理密钥时，可以将代理转发到远程服务器。这对于在远程服务器上进行进一步的 SSH 连接（跳板机）很有用，而无需在远程服务器上存储私钥。

6. **AllowTcpForwarding yes**:
   - **作用**: 允许 TCP 端口转发。
   - **使用场景**: 用于创建隧道以访问远程服务。常用于保护不安全的服务或在防火墙后访问内网资源。比如通过 SSH 隧道访问数据库或内部网页。

7. **GatewayPorts no**:
   - **作用**: 控制是否允许远程主机连接到本地转发的端口。
   - **使用场景**: 默认情况下，转发的端口只能被本地主机访问。设置为 `yes` 时，允许其他主机访问这些端口。这在需要将服务暴露给外部网络时有用，但有安全风险。

8. **X11Forwarding yes**:
   - **作用**: 启用 X11 转发。
   - **使用场景**: 允许在 SSH 会话中使用图形应用程序。对于需要从远程服务器运行图形界面的应用（如 GUI 编辑器或图形工具）非常有用。

9. **X11DisplayOffset 10**:
   - **作用**: 设置 X11 显示编号的偏移量。
   - **使用场景**: 用于避免与本地 X11 显示冲突。通常不需要更改，除非你有特定的显示编号需求。

10. **X11UseLocalhost yes**:
   - **作用**: 控制 X11 转发是否使用 localhost 地址。
   - **使用场景**: 设置为 `yes` 时，X11 转发使用 localhost 地址，增强安全性，防止外部访问。设置为 `no` 可以允许外部访问，但通常不推荐。

11. **PermitTTY yes**:
   - **作用**: 允许分配 TTY（终端）。
   - **使用场景**: 使用户能够在 SSH 连接中使用交互式终端。通常需要启用，以便用户可以执行命令和脚本。

12. **PrintMotd no**:
   - **作用**: 控制是否在登录时显示 `/etc/motd`（信息提示）。
   - **使用场景**: 如果不希望显示登录信息提示，可以设置为 `no`。

13. **PrintLastLog yes**:
   - **作用**: 显示用户上次登录的信息。
   - **使用场景**: 提供安全信息，帮助用户了解上次登录时间和来源。

14. **TCPKeepAlive yes**:
   - **作用**: 启用 TCP 保持活动状态。
   - **使用场景**: 检测死连接并保持连接活跃。对长时间运行的 SSH 会话有帮助。

15. **PermitUserEnvironment no**:
   - **作用**: 禁止用户通过 `.ssh/environment` 文件设置环境变量。
   - **使用场景**: 增强安全性，防止用户在登录时改变环境变量。

16. **Compression delayed**:
   - **作用**: 启用 SSH 连接的压缩。
   - **使用场景**: 在连接建立后启用压缩，节省带宽。适用于低带宽连接。

17. **ClientAliveInterval 30**:
   - **作用**: 设置服务器向客户端发送保持活动消息的时间间隔（秒）。
   - **使用场景**: 确保连接活跃，检测失效连接。

18. **ClientAliveCountMax 10**:
   - **作用**: 在客户端未响应时，服务器发送保持活动消息的最大次数。
   - **使用场景**: 控制连接失效前的最大尝试次数。

19. **UseDNS no**:
   - **作用**: 控制是否使用 DNS 解析客户端主机名。
   - **使用场景**: 设置为 `no` 可以加快连接速度，尤其是在 DNS 解析缓慢时。

20. **PidFile /var/run/sshd.pid**:
    - **作用**: 指定 SSH 守护进程的 PID 文件路径。
    - **使用场景**: 用于进程管理和监控。

21. **MaxStartups 10:30:100**:
    - **作用**: 控制并发未认证连接的数量。
    - **使用场景**: 防止 DoS 攻击，通过限制并发连接来保护服务器。

22. **PermitTunnel no**:
    - **作用**: 禁止 SSH 隧道。
    - **使用场景**: 增强安全性，防止通过 SSH 建立 VPN 或其他隧道。

23. **ChrootDirectory none**:
    - **作用**: 指定用户登录时的 chroot 目录。
    - **使用场景**: 用于限制用户访问范围，增强安全性。

24. **VersionAddendum none**:
    - **作用**: 控制 SSH 版本信息的附加内容。
    - **使用场景**: 隐藏版本信息以提高安全性。

25. **Banner none**:
    - **作用**: 控制登录时显示的横幅信息。
    - **使用场景**: 可以设置自定义横幅，或禁用默认横幅。

26. **AcceptEnv LANG LC_***:
    - **作用**: 允许客户端传递语言和区域设置环境变量。
    - **使用场景**: 使客户端能够在服务器上使用本地化设置。

27. **Subsystem sftp /usr/lib/openssh/sftp-server**:
    - **作用**: 定义 SFTP 子系统的位置。
    - **使用场景**: 启用 SFTP 功能，允许文件传输。
