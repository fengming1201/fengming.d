要使用 `mdadm` 配置三个硬盘为 RAID 5，您可以按照以下步骤进行：

### 1. 安装 mdadm
确保 `mdadm` 已安装。

```bash
sudo apt update
sudo apt install mdadm
```

### 2. 准备硬盘
确保三个硬盘没有重要数据，因为配置 RAID 会清除数据。

### 3. 创建 RAID 5 阵列
假设硬盘分别为 `/dev/sdb`、`/dev/sdc` 和 `/dev/sdd`。

```bash
sudo mdadm --create --verbose /dev/md0 --level=5 --raid-devices=3 /dev/sdb /dev/sdc /dev/sdd
```

### 4. 创建文件系统
在新创建的 RAID 设备上创建文件系统。

```bash
sudo mkfs.ext4 /dev/md0
```

### 5. 挂载 RAID 设备
创建一个挂载点并挂载 RAID 设备。

```bash
sudo mkdir -p /mnt/raid5
sudo mount /dev/md0 /mnt/raid5
```

### 6. 更新 /etc/fstab
为了在启动时自动挂载 RAID 设备，编辑 `/etc/fstab` 文件。

```bash
echo '/dev/md0 /mnt/raid5 ext4 defaults,nofail,discard 0 0' | sudo tee -a /etc/fstab
```

### 7. 保存 RAID 配置
保存 RAID 配置以便系统重启后识别。

```bash
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf
sudo update-initramfs -u
```

### 8. 检查 RAID 状态
可以使用以下命令检查 RAID 阵列的状态：

```bash
cat /proc/mdstat
sudo mdadm --detail /dev/md0
```

通过这些步骤，您可以成功配置三个硬盘为 RAID 5 阵列。确保在操作前备份重要数据。

在 RAID 5 阵列中，如果一个硬盘损坏，需要将其替换以恢复阵列的冗余性。以下是详细步骤：

### 1. 确认损坏的硬盘
首先，检查 RAID 阵列的状态，确认哪个硬盘损坏。

```bash
cat /proc/mdstat
sudo mdadm --detail /dev/md0
```

### 2. 标记损坏的硬盘为故障
假设损坏的硬盘是 `/dev/sdb`。

```bash
sudo mdadm --manage /dev/md0 --fail /dev/sdb
sudo mdadm --manage /dev/md0 --remove /dev/sdb
```

### 3. 物理更换硬盘
关闭系统，更换物理损坏的硬盘。更换完毕后，启动系统。

### 4. 添加新硬盘到 RAID 阵列
假设新硬盘被识别为 `/dev/sdb`，需要将新硬盘添加到 RAID 阵列中。

首先，确保新硬盘没有分区表或文件系统：

```bash
sudo fdisk /dev/sdb
```

在 `fdisk` 中：
- 输入 `o` 创建一个新的空的 DOS 分区表。
- 输入 `w` 写入更改并退出。

然后，将新硬盘添加到 RAID 阵列：

```bash
sudo mdadm --manage /dev/md0 --add /dev/sdb
```

### 5. 检查重建过程
RAID 阵列会自动开始重建。可以通过以下命令检查重建进度：

```bash
cat /proc/mdstat
```

### 6. 更新 mdadm 配置
确保 `mdadm` 配置文件是最新的，以便系统重启后识别新的 RAID 阵列配置。

```bash
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf
sudo update-initramfs -u
```

通过这些步骤，您可以成功替换 RAID 5 阵列中的损坏硬盘，并恢复阵列的冗余性。确保在操作前备份重要数据。
